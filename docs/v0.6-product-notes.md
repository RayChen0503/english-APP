# v0.5/v0.6 產品原型說明

## 這版解決什麼

前一版已經能跑出完整流程，但程式仍偏向單檔展示。v0.5/v0.6 把原型整理成可持續開發的產品骨架，讓後續要接 AI、資料庫、帳號或重新設計 UI 時，不需要從零開始。

## 架構

- `model`：定義 App 中穩定存在的概念，例如學生、心情、任務、斷點、週報。
- `data`：集中放展示用資料。未來可以替換成 Firebase、Room 或 API。
- `state`：保存本機互動狀態。現在使用 SharedPreferences，未來可改成 Room。
- `ui`：集中建立卡片、按鈕、文字樣式。未來若改設計系統，先從這裡調整。
- `MainActivity`：目前仍是單 Activity，但畫面流程已經和資料層分開。

## 展示亮點

1. 首頁不直接考試，先呈現學生狀態與今日任務。
2. 心情檢測會改變任務時間與模式。
3. 答題成功會增加微任務數與信心值。
4. 答錯會進入 AI 即時陪伴，不直接懲罰。
5. 連續答錯會生成斷點，並轉給老師/志工端。
6. 老師端可以查看學生列表、斷點摘要、陪伴腳本與週報。
7. 學習地圖用個人進度取代排行榜。
8. 完整使用旅程把「心情檢測 → 短任務 → AI 拆解 → 志工接力 → 週報回饋」串成可展示流程。
9. 主動求助流程讓學生能用自己的語句表達卡住原因，平台再做分流。
10. 產品設計原則頁可用於報告，說明每個功能如何回應課程提案。
11. 今日學習契約把任務邊界先講清楚，降低學生對「打開 app 就要被測驗」的焦慮。
12. 課後反思把任務完成後的感受轉成可被老師理解的修復證據。
13. 老師端待辦佇列把「誰負責、何時處理、下一步做什麼」具體列出來。

## 還沒做的正式功能

- 真實登入與多帳號。
- 真 AI API。
- 真實題庫與課綱同步。
- 後端資料庫。
- 推播通知。
- 老師端批次管理與資料匯出。

## 長工模式追加內容

### Phase 1：建置能力

- 確認 Gradle wrapper 可用。
- 改用 Android Studio 內建 JBR 執行命令列 build。
- `:app:assembleDebug` 已可成功執行。

### Phase 2：產品資訊架構

- 新增學生學習檔案。
- 新增今日任務佇列。
- 新增陪伴訊息時間線。
- 新增週訊號資料。

### Phase 3：學生端深度

- 新增復原任務，對應情緒低落或連續答錯時的低壓修復。
- 新增錯題修復紀錄。
- 新增離線任務包，對應偏鄉網路與碎片時間限制。

### Phase 4：老師/志工端深度

- 新增接力優先序。
- 新增 OPPM 品質檢核：差異化、動機導向、可執行性、低負擔、完整度。
- 老師端不只看資料，也能知道下一步誰負責、該怎麼陪伴。

### Phase 5：v0.6 產品流程補強

- 新增完整使用旅程頁，讓展示者能從學生心理狀態一路講到老師接力。
- 新增情緒斷點處理流程頁，把心情低落、連續答錯、需要真人、完成修復都對應到具體產品行為。
- 新增產品設計原則頁，讓 mentor 或老師可以檢查原型是否符合「低壓先行、AI 不取代真人、進步不靠排名、偏鄉限制內建」。
- 新增主動求助流程，學生點選不同求助原因後，會分別進入 AI 即時陪伴、復原任務、離線任務或志工接力。

### Phase 6：長工版產品閉環

- 新增今日學習契約，讓學生、平台、志工三方的任務邊界更清楚。
- 新增課後 20 秒反思，學生完成任務後可表達「更懂一點、仍然卡住、今天先到這裡」。
- 新增老師端待辦處理佇列，讓接力摘要不只停在資料展示，也能變成可執行工作。
- 新增離線/同步狀態，補上偏鄉網路不穩時原型需要呈現的產品回饋。

### Phase 7：1-6 大項本機可操作原型

- 真實資料功能：新增本機帳號中心，模擬學生、志工、老師登入狀態與班級/群組代碼。
- 真 AI 功能：新增 AI 提示實驗室，展示診斷、學生回饋、志工接力摘要三段式輸出。
- 老師/志工端：新增學生資料管理與待辦完成計數，老師端能模擬新增展示學生與標記處理進度。
- 學生端學習：新增閱讀拆句、聽力暖身、會考高頻字等題型，讓流程不只停在 be 動詞。
- UI 完整度：新增帳號、AI、同步、資料管理入口，讓產品架構更完整。
- 離線同步：新增同步中心，可模擬待上傳紀錄補傳與全部同步。

### Phase 8：可操作性補強

- 新增學生不只改數字，也會出現在老師端學生清單。
- 老師端待辦可逐項點擊完成，已完成項目會改成完成狀態。
- 志工接力頁可新增志工回覆，回覆數會保存並顯示。
- 學生答題、答錯、反思會累積學習事件、修復紀錄與待同步紀錄。
- 學習地圖新增時間線，能看到學習事件、錯題修復、待同步狀態。
- AI 生成頁會依題型給不同診斷，不再只是固定文案。

## 建議展示路線

1. 從學生端首頁開始，說明平台先看心情與時間，而不是直接考試。
2. 點「查看完整使用旅程」，介紹雙軌平台如何從學生挫折一路接到真人陪伴。
3. 回到任務，故意答錯一次，展示 AI 即時陪伴。
4. 點「我想直接求助」，選「我一直答錯」，展示學生主動求助和志工摘要。
5. 答對一題後點「做一個 20 秒反思」，展示平台如何保存修復證據。
6. 切到老師/志工端，展示工作台、待辦處理佇列、接力優先序、陪伴腳本與週報。
7. 最後打開「產品設計原則」或「OPPM 品質檢核」，把原型連回課程提案。
